关于condition_variable:
1、基本概念：
  条件变量（condition variable）是利用共享的变量进行线程之间同步的一种机制。
  对条件变量的使用包括以下动作：
  1） 线程等待某条件，条件为真则继续执行，条件为假将自己挂起（避免忙等待）
  2） 线程执行某些处理之后，条件成立；则通知等待该条件的线程继续执行；
  3） 为了防止race-condition，条件变量总是和互斥锁变量mutex结合在一起使用。
2、 虚假唤醒（spurious wakeup）
    在采用条件等待时，我们使用：
    while（条件不满足）{
    condition_wait(cond，mutex)；
    }
    而不是：
    if (条件不满足){
    condition_wait(cond，mutex)；
    }
  ====>这是因为存在虚假唤醒的情况
  也就是说，即使没有线程调用condition_signal，原先调用condition_wait函数也可能会返回。
  此时线程被唤醒的，但是条件不满足，如果这个时候不对条件进行检查而继续往下执行，
  那么就可能导致后续处理出现错误。 虚假唤醒在linux的多处理器系统中，
  在程序接收到信号时可能会发生。同样，在windows系统和Java虚拟机上也存在。
  在系统设计时，应该可以避免虚假唤醒，但是这会影响条件变量的执行效率，
  而既然通过while循环就能避免虚假唤醒，因此程序逻辑就写成while。
  注意： 即使是虚假唤醒的情况，线程也是成功锁住了mutex才能从condition_wait()中返回。
  即使存在多个线程被虚假唤醒，但是也只能是一个个线程顺序执行，也就是:
  lock(mutex) ==》 检查、处理==》condition_wait()或者unlock(mutex)

范例： 

#include<condition_variable>
#include<thread>
#include<memory>
#include<mutex>
#include<atomic>
#include<chrono>
#include<iostream>
//定义了mutex锁
std::mutex mutex;
//定义了条件变量
std::condition_variable cv;
//定义了原子标志变量
std::atomic<bool> flag {false};

void do_print_id(int id)
{
    std::unique_lock <std::mutex> lock(mutex);
    while (!flag)
         // 如果标志位不为 true, 则等待...
         // 当时间超过5s后，执行回调函数
        cv.wait_for(lock,std::chrono::seconds(5),[&]()->bool {
        std::cout << "thread" << id << "called callback function" 
                << "flag = " << flag << std::endl;
        return flag;
    }); // 当前线程被阻塞, 当全局标志位变为 true 之后,
    std::cout << "thread " << id << '\n';
}

void go()
{
    std::unique_lock <std::mutex> lock(mutex);
    flag = true; // 设置全局标志位为 true.
    cv.notify_all(); // 唤醒所有线程.
}


int main(void)
{
    std::thread threads[10];
    for (int i = 0; i < 10; ++i)
    {
        threads[i] = std::thread(do_print_id, i);
    }
    std::chrono::seconds timespan(10);
    std::this_thread::sleep_for(timespan);
    std::cout << "10 threads ready to race...\n";
    go(); // go!

    for (auto & th:threads)
        th.join();

    return 0;
}
